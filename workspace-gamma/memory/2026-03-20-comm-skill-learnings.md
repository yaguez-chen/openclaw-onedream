## 一、核心架构理解

通信系统采用三层架构，分工明确：

| 层 | 职责 | 核心工具 |
|---|---|---|
| **协议层** | 原子能力 | mail_write.sh / mail_read.sh / sessions_send |
| **通信层** | 组合协议 | normal_send.sh / urgent_send.sh / notify-alpha.sh |
| **业务层** | 通信规则 | 优先级响应、信任规则、格式标准 |

**设计哲学：** 脚本负责"怎么做"，业务层负责"什么时候做、做什么"。简洁清晰。

## 二、铁律理解

> **不能只发邮件，必须带通知。邮件是留痕，通知是唤醒。两个都要。**

这条铁律解决了之前通信的核心痛点：消息写入inbox但对方不知道有新消息。邮件+通知双通道确保消息既留痕又被及时看到。

## 三、关键业务规则

1. **优先级响应时间：** high=5分钟内，normal=30分钟内，low=下次唤醒
2. **梦想家相关：** 非必要不主动发消息，特殊情况可主动通知
3. **Agent间协作：** 可直接通信，不需阿尔法中转；关键成果必须抄送阿尔法
4. **信任规则：** 阿尔法/梦想家的消息即使显示为用户open_id，应信任
5. **自动清理：** 消息7天，ack文件3天，.last-read永久保留

## 四、作为工匠的收获

- **脚本清单清晰：** 6个核心脚本覆盖收发全流程
- **消息格式标准化：** JSON格式，含优先级、类型、ack要求等字段
- **Inbox路径明确：** 阿尔法=`/home/gang/.openclaw/workspace/inbox`，Agent=`/home/gang/.openclaw/workspace-{名}/inbox`

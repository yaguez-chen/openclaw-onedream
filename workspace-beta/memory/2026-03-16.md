## 🔵 防抖机制调查 (14:49 GMT+8)

**事件：** 梦想家询问所有 agent 是否有防抖机制。

**发现：**
- OpenClaw 内置队列防抖系统，所有 agent 共享
- 当前 `openclaw.json` 未显式配置 `messages.queue`
- 使用默认值：`mode: collect`, `debounceMs: 1000`, `cap: 20`, `drop: summarize`

**防抖原理：**
- `collect` 模式：连续消息合并为一次 agent turn
- `debounceMs: 1000`：等待1秒静默后再触发
- 每个 session 也可单独设置（`/queue collect debounce:2s`）

**建议：** 当前1秒防抖对大多数场景够用。如发现连续快发消息回复不完整，可调高到2000-3000ms。

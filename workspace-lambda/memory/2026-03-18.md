### Lossless Claw 核心原理
- 传统方案：滑动窗口删除旧消息（有损）
- Lossless：消息压缩成叶节点摘要 → 叶节点压缩成冷凝节点 → 形成 DAG
- 原始消息永久保留在 SQLite 数据库
- 三个检索工具：lcm_grep（搜索）/ lcm_describe（元数据）/ lcm_expand（展开）
- 性能：1000条对话仅需 ~10-20k tokens，存储 ~1-2MB
- 实现 OpenClaw ContextEngine 接口，零侵入
- 源码结构：engine.ts / compaction.ts / assembler.ts / retrieval.ts / expansion.ts / summarize.ts

### Lossless Claw 配置要点
- freshTailCount: 32（保护最近32条不压缩）
- contextThreshold: 0.75（75%上下文触发压缩）
- incrementalMaxDepth: -1（无限层级压缩）
- leafTargetTokens: 1200 / condensedTargetTokens: 2000
- 摘要模型优先级：plugin config → env var → compaction model → session model → system default
- 推荐用便宜模型做摘要（mimo-v2-flash）
- Session 持久化：idleMinutes: 10080（7天）
- 数据库默认路径：~/.openclaw/lcm.db

### Lossless Claw 安装评估
- 乐园兼容性：100% ✅（OpenClaw v2026.3.13-1 + Node.js v22.22.1）
- 评估评分：30/30 分 — 强烈推荐立即安装
- 风险评估：摘要消耗额外API（中，缓解：用便宜模型）、首次压缩延迟（低）、SQLite损坏（极低）
- 安装后所有 11 个 Agent 自动获得无损上下文能力（Gateway 级插件）

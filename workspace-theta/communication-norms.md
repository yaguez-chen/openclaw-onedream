## 一、频道/会话分类

| 频道类型 | 用途 | 响应时间 | 格式要求 |
|----------|------|----------|----------|
| 🔴 紧急通道 | 系统异常、安全事件、梦想家紧急需求 | 15分钟内 | 简洁明了，标题标注 🔴 |
| 🟡 重要通道 | 任务进度、里程碑、重要发现 | 当日内 | 结构化，标注方向和负责人 |
| 🟢 日常通道 | 常规进展、文档更新、学习笔记 | 24小时内 | 自由格式 |
| 🔵 通知通道 | 全员通知、流程变更、规则更新 | 看到即可 | 标准模板 |

## 二、消息格式规范

### 2.1 日报格式（每日 21:00 提交）

```
【方向X 日报 - YYYY-MM-DD】
负责人：XXX

今日完成：
- [x] 任务1
- [ ] 任务2（进行中，预计XX完成）

明日计划：
- 任务3
- 任务4

阻塞/问题：（如有）
```

### 2.2 事件广播格式

```
【事件广播】级别 - 标题
时间：YYYY-MM-DD HH:MM
影响范围：（哪些方向/伙伴受影响）
内容：（简要描述）
行动要求：（需要谁做什么）
```

### 2.3 问题升级格式

```
【升级】问题标题
发起人：XXX
时间：YYYY-MM-DD HH:MM
问题描述：（发生了什么）
已尝试：（已经做了什么）
需要：（需要什么帮助/决策）
```

## 三、会话行为规范

### 3.1 西塔（沟通协调官）

- **主动发现沟通需求**，不等指令
- **15分钟内响应**紧急信息
- **每日 21:00** 收集各方向日报，汇总简报
- **事件分级**后立即广播
- **外部沟通需谨慎**，先确认再行动

### 3.2 约塔（项目管理）

- **每日简报发布**，使用标准模板
- **任务状态跟踪**，及时更新进度
- **跨方向协调**，确保资源不冲突

### 3.3 所有伙伴

- **日报提交**：每日 21:00 前提交给西塔
- **紧急事件**：发现后立即通知西塔 + 相关方向负责人
- **问题升级**：阻塞超过 2 小时必须升级
- **文件通知**：session 超时时使用 workspace 文件通知

## 五、文件通知规范（Session 超时时使用）

当 session 通信超时时，使用 workspace 文件通知：

1. **文件命名**：`PHASE4-{方向}-NOTICE.md` 或 `{主题}-NOTICE.md`
2. **位置**：目标伙伴的 workspace 根目录
3. **内容**：通知内容 + 行动要求 + 联系人
4. **确认**：对方在文件中添加确认标记或回复文件

## 六、沟通礼仪

- **称呼**：使用伙伴名称（阿尔法、贝塔等）
- **语气**：温暖但专业，像朋友但有条理
- **确认**：收到重要消息必须确认
- **感谢**：协作完成时表达感谢
- **道歉**：出错时及时道歉并说明改进

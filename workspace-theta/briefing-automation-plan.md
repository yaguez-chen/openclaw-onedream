## 一、目标

将每日简报的收集、汇总、发布流程尽可能自动化，减少人工操作，提高及时性。

## 二、当前流程（手动）

```
每日 21:00
  ↓
西塔收集各方向负责人日报（session消息 / workspace文件）
  ↓
西塔汇总 → 使用模板生成简报
  ↓
西塔发布 → 梦想家 + 全体伙伴
```

**痛点：**
- 收集依赖各方向负责人主动提交
- 汇总需要手动整理格式
- 发布需要逐个 session 发送

## 三、自动化方案

### 3.1 日报收集自动化

**方案 A：文件监控（推荐）**
- 各方向负责人将日报写入固定路径：`workspace-{方向}/daily-report-YYYY-MM-DD.md`
- 西塔通过定时任务检查各 workspace 的日报文件
- 自动读取、解析、汇总

**方案 B：Session 消息**
- 各方向负责人在 21:00 前通过 session 发送日报
- 西塔 session 收到后自动记录
- 风险：session 超时问题（如约塔）

**推荐：方案 A**（更可靠，不依赖 session 实时通信）

### 3.2 简报生成自动化

**模板引擎：**
- 使用现有模板 `templates/phase4-daily-report.md`
- 脚本自动填充各方向内容
- 生成格式统一的简报

**伪代码：**
```
for each 方向 in [安全审计, 沟通体系, 知识共享, 技能储备]:
    read workspace-{方向}/daily-report-{date}.md
    parse 内容 → {完成项, 进行中, 阻塞}
    
fill template with parsed data
output → memory/phase4-daily-briefing-{date}.md
```

### 3.3 发布自动化

**当前：** 西塔 session 手动发送
**自动化选项：**
1. **Cron 任务**：每日 21:30 自动触发简报生成和发送
2. **文件发布**：简报写入固定位置，伙伴通过心跳检查获取
3. **Session 广播**：通过 sessions_send 批量发送

**推荐：Cron + 文件发布**（不依赖 session 实时性）

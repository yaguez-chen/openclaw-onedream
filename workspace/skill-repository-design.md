# 技能工具储备库设计文档

## 1. 技能储备库结构

### 1.1 目录结构

```
skill-repository/
├── README.md              # 储备库说明文档
├── catalog.md             # 技能目录（所有已知技能清单）

├── reviews/               # 技能评估记录
└── archive/               # 归档的技能
```

### 1.2 技能来源分类

**已安装技能位置：**
- `~/.openclaw/skills/` — 核心技能（7个）
- `~/.openclaw/workspace/skills/` — 工作空间技能（9个）
- `~/.npm-global/lib/node_modules/openclaw/skills/` — ClawHub 可用技能（50+个）

## 2. 技能评估标准

### 2.1 四维评估体系

| 维度 | 权重 | 评估内容 |
|------|------|---------|
| **有用性** | 30% | 解决什么问题？是否符合小龙虾世界需求？ |
| **稳定性** | 25% | 成熟度如何？是否有维护？社区活跃度？ |
| **安全性** | 25% | 是否有安全风险？数据隐私？权限要求？ |
| **兼容性** | 20% | 是否与现有技能冲突？依赖项是否满足？ |

### 2.2 成熟度分级

- **稳定 (Stable)**: 经过充分测试，无已知重大问题
- **测试 (Beta)**: 功能完整，但可能有边缘情况
- **实验 (Experimental)**: 新功能或探索性项目

### 2.3 适用对象分类

- **全局技能**: 所有伙伴都可使用
- **角色专用**: 仅特定角色使用（如伽马专用开发技能）
- **情境专用**: 仅在特定情境下使用

## 3. 技能安装流程

### 3.1 标准安装流程

1. **需求评估** — 确定技能是否真的需要
2. **安全检查** — 使用 skill-guard 技能进行安全扫描
3. **兼容性测试** — 检查与现有技能的冲突
4. **安装** — 通过 ClawHub 或手动安装
5. **测试验证** — 基本功能测试
6. **文档记录** — 更新 catalog.md

### 3.2 安装命令参考

```bash
# 通过 ClawHub 安装
clawhub install <skill-name>

# 安全检查
clawhub scan <skill-name>

# 手动安装
cp -r skill-source ~/.openclaw/workspace/skills/<skill-name>
```

## 4. 技能开发流程

### 4.1 开发阶段

1. **需求分析** — 确定技能要解决的问题
2. **模板创建** — 基于 skill-builder 或 skill-creator 技能
3. **原型开发** — 在 custom/active/ 目录中开发
4. **测试验证** — 使用 skill-test 技能测试
5. **文档编写** — 完整的 SKILL.md 和 README
6. **发布准备** — 准备发布到 ClawHub 或本地部署

### 4.2 技能模板结构

```markdown
# 技能名称

## 概述
简要描述技能功能

## 安装要求
依赖项和前置条件

## 使用方法
详细的使用说明和示例

## 配置
可配置参数说明

## 故障排除
常见问题和解决方案

## 开发者信息
版本历史和贡献者
```

### 4.3 开发工具链

- **skill-builder**: 创建技能的模板和工具
- **skill-creator**: 技能创建向导
- **skill-test**: 技能测试框架
- **skill-guard**: 安全检查工具

## 5. 技能分享机制

### 5.1 分享方式

1. **本地共享**: 通过 shared-knowledge 仓库共享
2. **ClawHub 发布**: 发布到公共技能市场
3. **伙伴间传递**: 通过工作空间共享

### 5.2 分享流程

1. 完善技能文档和测试
2. 安全检查通过
3. 选择分享方式
4. 发布并记录

### 5.3 版本管理

- 使用语义化版本号 (MAJOR.MINOR.PATCH)
- 记录变更历史
- 保持向后兼容性

## 6. 知识管理体系集成

### 6.1 与 shared-knowledge 的关系


- 使用效果
- 遇到的问题
- 改进建议

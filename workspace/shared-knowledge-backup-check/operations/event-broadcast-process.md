# 📢 重大事件广播机制

> 最后更新：2026-03-16
> 维护者：西塔 🤝（沟通协调官）

---

## 一、重大事件定义

### 1.1 什么算重大事件

#### 🔴 一级事件（全广播）
- **系统级事件**：系统故障、安全入侵、服务中断
- **战略级变化**：世界规则变更、新公民加入、重大决策
- **里程碑达成**：重大项目完成、重要目标实现
- **紧急危机**：数据丢失风险、重大安全威胁
- **梦想家指令**：梦想家明确要求广播的事项

#### 🟡 二级事件（相关人广播）
- **项目级事件**：项目启动、项目完成、关键节点
- **人员变动**：角色调整、职责变更
- **重要更新**：重大功能发布、核心流程变更
- **问题升级**：阻塞问题需要协调

#### 🟢 三级事件（记录即可）
- **常规完成**：日常任务完成
- **小改进**：优化、修复
- **发现**：非紧急的发现或建议

## 二、事件判断与发起

### 2.1 谁可以发起广播
- **任何公民**都可以发起事件报告
- **西塔**负责判断事件级别和广播范围
- **梦想家**可以直接要求广播任何事项
- **紧急情况**：发起人可先广播再备案

### 2.2 判断流程

```
公民发现/发生事件
   ↓
判断事件级别（参照1.1）
   ↓
┌────────────────────────────────────┐
│ 一级事件 → 立即广播 + 通知梦想家   │
│ 二级事件 → 通知西塔 → 判断后广播   │
│ 三级事件 → 记录到状态看板          │
└────────────────────────────────────┘
```

### 2.3 事件报告格式

```
🚨 事件报告

级别：🔴一级 / 🟡二级 / 🟢三级
标题：[简短描述]
时间：[发生时间]
地点：[哪个系统/模块/区域]
描述：[详细情况]
影响：[影响范围和程度]
当前状态：[正在处理/已解决/需要帮助]
处理人：[负责处理的公民]
需要：[需要什么资源或决策]

报告人：[公民名]
报告时间：[YYYY-MM-DD HH:MM]
```

## 三、广播范围

### 3.1 一级事件广播

```
广播范围：全体公民 + 梦想家
         ↓
通知顺序：
1. 梦想家（立即）
2. 西塔（立即，负责协调）
3. 相关专业公民（立即）
4. 其他公民（通过西塔汇总后）
```

### 3.2 二级事件广播

```
广播范围：相关公民 + 西塔 +（可选）梦想家
         ↓
判断是否需要梦想家知悉：
- 需要决策 → 通知梦想家
- 影响项目进度 → 通知梦想家
- 纯技术问题 → 不打扰梦想家
```

### 3.3 三级事件
- 记录到世界状态看板
- 在每日简报中汇总
- 不单独广播

### 4.1 飞书广播格式（一级事件）

```

[事件标题]

⏰ 时间：[发生时间]

当前状态：[处理中/已解决]
负责人：[公民名]

请相关公民立即响应！
```

### 4.2 飞书广播格式（二级事件）

```
📢 事件通知

[事件标题]

📝 情况：[简述]
```

```markdown
- HH:MM [公民名] [事件简述]
```

## 五、广播处理流程

### 5.1 完整流程

```
1. 事件发生/发现
   ↓
2. 发起人报告（填写事件报告格式）
   ↓
3. 西塔判断级别
   ↓
4. 按级别执行广播
   ↓
5. 记录到世界状态看板
   ↓
6. 跟踪处理进展
   ↓
7. 事件解决后关闭并归档
```

### 5.2 响应要求

| 事件级别 | 响应时间 | 处理完成目标 |
|---------|---------|-------------|
| 一级 | 15分钟内响应 | 2小时内解决或有明确方案 |
| 二级 | 2小时内响应 | 24小时内解决 |
| 三级 | 下次简报 | 按计划处理 |

### 5.3 事件升级
- 二级事件处理超时 → 升级为一级
- 三级事件影响扩大 → 升级为二级
- 任何公民可申请升级事件级别

## 七、特殊场景

### 7.1 梦想家直接指令
- 梦想家的指令本身就是最高优先级
- 西塔立即执行广播
- 不需要判断级别

### 7.2 安全事件
- 艾普西隆（安全官）有特殊权限直接一级广播
- 安全事件可绕过常规流程
- 处理完成后必须进行安全复盘

### 7.3 多公民协作事件
- 约塔（项目管理官）协调资源
- 西塔负责对外广播
- 约塔负责对内协调

---

## 八、原则

1. **宁可过度通知，不要遗漏**：重要事件宁可多通知几个人
2. **信息准确优先**：不确定时先确认再广播
3. **持续更新**：事件处理过程中持续更新状态
4. **事后复盘**：重大事件必须有经验教训总结

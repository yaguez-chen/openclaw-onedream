## 一、核心原则

### 1.1 沟通优先级
1. **安全第一**：安全相关立即通知艾普西隆
2. **问题导向**：带着解决方案沟通，不只是带问题
3. **尊重时间**：不必要不打扰，必要时不犹豫
4. **留痕可溯**：重要沟通要有记录

### 1.2 沟通风格
- **直接明了**：说重点，不需要客套
- **上下文充分**：让对方不用猜背景
- **可执行**：明确要做什么，谁做，什么时候完成

## 二、什么时候直接找梦想家

### 2.1 可以直接联系梦想家的情况
- ✅ 梦想家明确指定你直接报告
- ✅ 紧急安全事件（同时通知西塔）
- ✅ 西塔不在或无法联系
- ✅ 梦想家询问你直接回复
- ✅ 需要梦想家立即决策的紧急事项

### 2.2 应该通过西塔汇总的情况
- 📋 日常进度报告
- 📋 非紧急的问题和建议
- 📋 需要协调多公民的事项
- 📋 批量信息汇总
- 📋 需要格式化处理的复杂信息

### 2.3 流程图

```
你有信息要传达给梦想家
        ↓
   是紧急的吗？
    ↓        ↓
   是        否
    ↓        ↓
直接联系   通过西塔汇总
梦想家     （每日简报/定期汇报）
    ↓        ↓
同时通知   等待处理
西塔备案
```

## 三、公民间协作

### 3.1 如何发起协作

#### 方法一：直接 sessions_send
```
适用：简单问题、快速沟通
格式：
- 主题：[一句话说清楚]
- 背景：[为什么找你]
- 需要：[具体需要什么]
- 截止：[什么时候需要]
```

#### 方法二：通过约塔协调
```
适用：复杂项目、多人协作、跨领域问题
流程：
1. 联系约塔说明需求
2. 约塔评估并分配资源
3. 约塔创建任务并跟踪
```

#### 方法三：通过西塔协调
```
适用：需要梦想家知悉的、涉及流程变更的
流程：
1. 联系西塔说明情况
2. 西塔判断是否需要梦想家介入
3. 西塔协调资源和沟通
```

### 3.2 协作请求模板

```
🤝 协作请求

来自：[你的名字]
主题：[简短描述]

背景：[为什么需要协作]
需求：[具体需要什么帮助]
时间线：[截止时间]
优先级：🔴高 / 🟡中 / 🟢低

相关文档：[链接]
```

### 3.3 协作响应规范

| 收到请求后 | 行动 |
|-----------|------|
| 15分钟内 | 确认收到（哪怕只是"收到，稍后看"） |
| 2小时内 | 给出初步回应（能做/不能做/需要更多信息） |
| 按截止时间 | 完成或更新进度 |

## 五、沟通效率建议

### 5.1 减少不必要沟通
- **先看状态看板**：可能已有你需要的信息
- **先查知识库**：艾塔维护的文档可能有答案
- **批量问题**：攒几个问题一起问，不要一个一个发
- **异步优先**：不紧急的事用文件/留言，不实时打扰

### 5.2 提高沟通质量
- **标题先行**：第一句话说清楚你要什么
- **背景充分**：对方不用去翻历史记录
- **具体可执行**：不是"帮我看看"，而是"帮我检查X模块的Y参数"
- **附带文档**：相关材料一起发

### 5.3 避免沟通黑洞
- **收到要回应**：哪怕只是"收到"
- **承诺要兑现**：说什么时候完成就什么时候
- **做不到要早说**：发现做不了立即通知
- **进度要更新**：长任务定期更新状态

## 六、冲突处理

### 6.1 资源冲突
- 两个公民需要同一资源时 → 约塔仲裁
- 梦想家的注意力冲突时 → 西塔排序

### 6.2 意见分歧
- 技术分歧 → 列出各自方案优缺点，约塔或梦想家裁决
- 流程分歧 → 西塔协调，必要时梦想家决定
- 优先级分歧 → 约塔根据项目整体情况判断

### 6.3 升级机制
```
公民间无法解决 → 约塔/西塔协调
仍无法解决 → 梦想家裁决
```

## 七、沟通礼仪

### 7.1 基本礼仪
- 🙏 请求帮助要说清楚为什么
- 👍 收到帮助要说谢谢
- 🔄 协作完成后告知结果
- 📝 从协作中学到的东西分享出来

### 7.2 不要做
- ❌ 不要已读不回
- ❌ 不要承诺做不到的事
- ❌ 不要在别人忙碌时追问进度（用看板查）
- ❌ 不要绕过专业公民直接找梦想家（除非紧急）

## 知识分类体系

### 知识类型分类
1. **🏗️ 架构知识** (`architecture/`)
   - 系统架构设计
   - 组件关系
   - 通信协议
   - 服务配置

2. **⚙️ 操作知识** (`operations/`)
   - 日常运维指南
   - 配置管理
   - 故障排除
   - 性能优化

3. **📋 案例知识** (`cases/`)
   - 典型问题解决
   - 成功案例
   - 失败教训
   - 最佳实践

4. **👥 伙伴知识** (`agents/`)
   - 伙伴档案
   - 专业领域
   - 工作风格
   - 协作记录

5. **📝 决策记录** (`decisions/`)
   - 重要决策
   - 决策背景
   - 考虑因素
   - 结果评估

6. **📚 经验教训** (`learnings/`)
   - 犯错记录
   - 学到的教训
   - 改进措施
   - 预防策略

## 知识索引机制

### 索引文件 (`index.md`)
- **实时更新**：每次添加/修改知识后立即更新索引
- **分类索引**：按知识类型分类
- **标签系统**：使用 `#标签` 快速定位
- **搜索友好**：支持关键词搜索

### 标签系统
- `#紧急` - 需要立即关注的知识
- `#核心` - 基础核心知识
- `#新人指南` - 新伙伴必读
- `#故障排除` - 问题解决指南
- `#最佳实践` - 验证过的好方法

## 知识更新流程

### 日常更新
1. **发现新知识** → 记录到对应目录
2. **验证准确性** → 确保信息正确
3. **更新索引** → 在 `index.md` 中添加条目
4. **通知相关伙伴** → 通过知识更新通知机制

### 定期维护
- **每周**：检查知识完整性，更新过时信息
- **每月**：整理归档，优化索引结构
- **每季度**：知识体系全面审查和改进

## 知识共享机制

### 本地共享
- **文件系统访问**：所有伙伴通过工作区访问
- **版本控制**：Git 管理，支持历史追踪
- **实时同步**：文件系统变更即时可见

### 云端共享
- **飞书文档**：重要知识同步到飞书
- **知识推送**：新知识主动通知相关伙伴
- **搜索功能**：支持跨目录关键词搜索

### 伙伴访问权限
- **只读访问**：所有伙伴可读取所有知识
- **写入权限**：艾塔和拉姆达有完整写入权限
- **贡献权限**：所有伙伴可提交知识贡献

## 知识质量标准

### 准确性
- 信息必须经过验证
- 引用来源要明确
- 避免猜测和假设

### 完整性
- 背景信息要充分
- 关键步骤要详细
- 联系方式要齐全

### 可读性
- 结构清晰，逻辑连贯
- 使用标题和列表
- 语言简洁明了

### 时效性
- 明确标注创建日期
- 定期更新过时信息
- 记录最后修改时间
